#+TITLE: Middleware Surface
#+AUTHOR: Darwin D. Wu
#+DATE: 2026-04-20
#+STATUS: Draft

* Goal
- Add an idiomatic Rust middleware surface with the required lifecycle ordering and mutation points.

* Scope
- Client middleware
- Function middleware
- Event send lifecycle hooks
- Remaining introspection capability reporting for advanced execution features

* Checklist
- [ ] Define an idiomatic Rust middleware API for client-level middleware
- [ ] Define an idiomatic Rust middleware API for function-level middleware
- [ ] Implement the required function-run lifecycle ordering
- [ ] Add transform-input hooks
- [ ] Add after-memoization hooks where supported
- [ ] Add before-execution hooks
- [ ] Add after-execution hooks
- [ ] Add transform-output hooks
- [ ] Add before-response hooks where supported
- [ ] Add event-send lifecycle hooks
- [ ] Allow middleware to mutate context, memoized step data, output, errors, and outbound events
- [ ] Add tests for middleware ordering and mutation points
- [ ] Extend probe/introspection capability reporting when advanced parallel execution features ship

* Exit Criteria
- [ ] Middleware exists with the required lifecycle guarantees
- [ ] Middleware behavior is test-covered
- [ ] Capability reporting is updated to reflect any newly shipped advanced execution features

* Out of Scope
- Connect implementation details, covered by `007`
- Checkpoint transport, covered by `007`
