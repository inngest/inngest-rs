#+TITLE: Remaining Step-Type Conformance
#+AUTHOR: Darwin D. Wu
#+DATE: 2026-04-20
#+STATUS: Active

* Goal
- Finish the step-type gaps that remain after `run`, `sleep`, `wait_for_event`, `invoke`, and durable event send landed.
- Finish the next parallel step-surface features deferred from `003`.

* Scope
- Invoke payload conformance cleanup
- Advanced parallel step APIs
- Parallel opcode metadata parity
- AI/gateway step types
- Deferred step-type documentation

* Checklist
- [X] Add a durable `step.send_event` or equivalent API
- [X] Decide whether send-event is implemented as a dedicated opcode or a wrapped run step
- [X] Ensure send-event behavior is retriable and memoized
- [X] Return sent event IDs in a useful developer-facing form
- [ ] Align invoke payload shape with the spec
- [ ] Revisit invoke timeout and payload serialization behavior
- [ ] Add a public `inngest::group::race!` API
- [ ] Decide and document the Rust-native relationship between `parallel!` and `race!`
- [ ] Emit explicit parallel-mode metadata where required for cross-SDK parity
- [ ] Add tests for `race!` discovery and targeted replay behavior
- [ ] Add tests for parallel-mode metadata in multi-step responses
- [ ] Decide and document the support plan for AI/gateway step types
- [ ] Implement any tractable AI/gateway step types that fit the existing runtime model, or explicitly document why they remain deferred
- [X] Add tests for durable event sending
- [X] Add tests for invoke memoization and error handling
- [ ] Document any intentionally deferred step types

* Exit Criteria
- [X] The core step API surface is no longer missing durable event send support
- [ ] Invoke behavior is spec-compatible
- [ ] The advanced parallel API surface beyond default `parallel!` is defined and tested
- [ ] Multi-step parallel responses carry any required mode metadata
- [ ] AI/gateway step-type support is either implemented or explicitly documented
- [ ] Deferred step types are explicitly documented instead of silently absent

* Out of Scope
- Connect, streaming, and checkpointing, covered by `007`
- Middleware, covered by `006`
